# stock-portfolio-analyzer

保有銘柄のポートフォリオを分析し、テクニカル・ファンダメンタル両面から評価スコアと売買の目安を表示するツールです。

## 現在のフェーズ

**フェーズ21: 最適化AI(情報量維持で圧縮)**
- 最終出力(通知テキスト/Discord embed)を、**情報量・数値・★・誠実ラベル(推定/検証/留意等)を保ったまま**圧縮する `optimize.py`。
- 冗長表現の短縮(現在かなり割安→割安)・重複行/同一理由の削除・句点の箇条書き化(。→・)・無駄な接続詞/あいまいな推測表現/丁寧表現(です/ます)の除去。文字数・トークン数・処理時間を削減し、要約末尾に「⚙️ 最適化: N字→M字(-X%)」を表示。
- 既に簡潔で20%削減できない場合はそのまま(品質最優先。**分析品質は落とさない**)。あわせて配分統計の `+nan%` 表示バグ(NaN混入)を修正。

**フェーズ20: 自己改修AI(レビュー反映→最終分析)**
- レビューAI(②)の指摘を分析結果へ**100%反映**し、最終分析を生成する `self_improve.py`。
- 方向性は維持し矛盾のみ修正: RR<1なのに買い→様子見、割高なのに買い→一段弱める、期待マイナスなのに買い→保有、確度が低い「強く買い増し」→一段下げ、NISA売却→温存(保有)、過大な期待リターン(>40%)→上限へ再計算、100点が出すぎ→抑制(序列は維持)。**点数・売買判定を再計算し、理由を追加・古いコメントを刷新**。
- 変更後は配分・リバランス・結論・レビューを**再計算**し、`レビュー→改修→再レビュー(改善不要)`のループを1実行内で閉じる。Discordに「🔧 自己改修(レビュー反映済み)」を表示。
- 無料のルールベースが既定。`ANTHROPIC_API_KEY` があれば Claude 版(`llm_revise`、提供プロンプトそのまま)も利用可(任意)。

**フェーズ19: iPhoneからの無償改修 + レビューAI(自己点検)**
- **iPhoneだけで無償運用・改修**: しきい値は `stock_analyzer/data/tuning.json`(1ファイル)で上書き可能。GitHubアプリ/github.dev で編集→コミット→次回実行に反映。手順は [`MOBILE.md`](MOBILE.md)。
- **レビューAI(`review.py`)**: 分析結果を鵜呑みにせず、第三者視点で品質を点検。ロジック矛盾(割高なのに買い/RRと判断の不一致/スコアと判断の食い違い)・リスク(決算跨ぎ)・ポートフォリオ(セクター偏り/NISA売却)・説明性・スコア(100点出すぎ/期待値過大)を検出し、各指摘に「なぜ問題か」「どう直すか」を付ける。問題無しなら「改善不要」。無料の**ルールベース**が既定で通知に載る。`ANTHROPIC_API_KEY` を設定すると**Claude によるレビュー**(提供プロンプトそのまま)も添付(任意・従量)。

**フェーズ18: Discordで銘柄コードを入力して分析(Cloudflare Worker)**
- Discord で `/analyze code:7203` と入力すると、その銘柄をAIファンドマネージャーで分析し、結果をチャンネルに返す。
- Cloudflare Worker(無料・サーバー管理不要・常時稼働)が Discord のスラッシュコマンドを受け、GitHub Actions を銘柄指定(`workflow_dispatch`)で起動 → 既存の Discord 通知として結果が届く。PCを閉じていても動作。
- セットアップ手順・コードは [`discord-bot/`](discord-bot/README.md) を参照。

**フェーズ17: 監視銘柄 + スイング候補の本格分析(TOPIX500ファネル)**
- **監視銘柄**: スプレッドシート/CSV で数量を空欄にした銘柄は「監視銘柄(未保有)」として、保有株と同じロジックで分析・表示する(🔍監視ラベル付き。保有損益・税は付かない)。保有していない気になる銘柄を並べておくと毎日分析される。
- **スイングTOP3を本格分析化**: 従来のテクニカル中心スクリーナーから、保有株と同じ判断ロジック(適正株価・割安率・成長・期間別期待リターン)へ変更。「今買うとしたら割安×成長で有望な順」の上位3銘柄を表示。
- **切り分け(ファネル)で多数を分析**: 対象を **TOPIX500**(TOPIX Core30+Large70+Mid400、約493銘柄)へ拡大。全銘柄を高速スクリーニング(価格のみ・分割ダウンロード)で絞り、上位のみファンダ取得を伴う本格分析にかけることで、取得回数・実行時間・レート制限を抑える。対象リストは `scripts/build_universe.py` が JPX 公式データから `stock_analyzer/data/universe.txt` を生成(定期的に再実行して更新可能。無ければ日経225にフォールバック)。

**フェーズ16: AIファンドマネージャー化(ポートフォリオ最適化 + 税効率 + 本日の結論)**
- 「分析ツール」から「AIファンドマネージャー」へ。裏では全指標を評価し、画面は結論だけを最小表示。
- **本日の結論(3行以内)**: 「今日買う/今日売る/現金比率の推奨」を最優先で先頭表示。売買すべき銘柄も是正すべき偏りも無いときは **「今日は何もしない」を能動的な最適判断**として明示(売買コスト・税・回転率の抑制)。
- **税効率最適化**: 保有に `account` 列(NISA/特定)を追加。含み損益(円)から、NISAは非課税で「枠を温存」(売却を後ろ倒し)、特定は約20.315%課税・含み損は損益通算に利用・利益20万円以下は売却しやすい、を注記として提示。取れない情報(他銘柄の実現損益など)は前提にせず捏造しない。
- **リバランス(現在比率→推奨比率)**: 保有の評価額比率を、魅力度(スコア×期待値)と銘柄/セクター上限から算出した推奨比率へ寄せる差分を、概算売買株数つきで提示(例: 40%→25%は縮小)。
- **市場環境の5段階化**: 強気/やや強気/中立/やや弱気/弱気に分類(株価指数の騰落数と平均変化率 + VIX補正)。米10年国債利回り(長期金利)も表示。騰落レシオ・新高値新安値・セクターローテーションは yfinance で取れないため加味しない(捏造しない)。
- 買い優先順位・次の投資資金の配分・想定配当利回り・期待リターン・想定リスク・分散評価をまとめた「AIファンドマネージャー判断」を表示。
- 期間別の期待リターン(1週/1ヶ月は検証済みバックテスト分布、半年〜1年はモデル推定)を各銘柄に表示。**誠実ラベル厳守**(スコアは「帯間の勝率差は小さい」但し書き、半年〜1年は「モデル推定」明示、需給★はプロキシと明記)。

- 日経平均・TOPIX・NYダウ・NASDAQ・S&P500・ドル円・VIXの現在値と前日比
- 上昇銘柄数と下落銘柄数の比較による市場全体の強気/弱気判定
- 各銘柄のセクター・業種を表示

**フェーズ11: スイング候補スクリーナー**
- 保有していない銘柄も対象に、日経225(約230銘柄)をスキャンし、スイング(数日〜2週間)向きの条件が揃った上位3銘柄を提示(1️⃣2️⃣3️⃣)
- 保有銘柄の通知は総合スコアの高い順に並び替え、評価に応じた色付き絵文字(🔴🟠🟡⚪🔵⚫)で銘柄ごとの違いを一目で判別可能
- 株価履歴は一括ダウンロードで高速取得し、テクニカル中心のスコアリング(MACD・25日線・出来高トレンド・レジスタンス突破・RSIの過熱手前など)で採点
- **これは「確実に上がる保証」ではなく、機械的スコアによる候補の絞り込みです。** 通知にも注意書きを表示。対象銘柄リストは `stock_analyzer/data/nikkei225.txt` で編集可能
- 廃止・データ欠損の銘柄は自動でスキップ
- 表示スコアは0〜100点で頭打ちですが、**順位付けは頭打ちしない内部の生スコア**で行うため、表示上100点で並んでも強さの順に正しく並びます

**フェーズ15: Discord通知対応(通知上限なし)**
- LINEに加えてDiscordのWebhook通知に対応。`DISCORD_WEBHOOK_URL` を設定すると、色付きのEmbedカードで通知(実質無制限・過去分の参照も快適)
- LINE・Discordの両方、またはどちらか一方に送信可能。分析データは1回だけ取得して各媒体に描画(無駄な再取得なし)

**フェーズ14: ファンダメンタルズ分析の拡充**
- 従来のPER・PBR・ROE・増収増益に加え、**ROA(資産効率)・配当利回り(株主還元)・財務健全性(負債資本倍率・流動比率)**を総合スコアに反映
- 流動比率1.0未満は「注意点」としても表示
- 詳細レポートに配当利回り・負債資本倍率・流動比率を追加表示

**フェーズ13: LINE Flex Message(カード型リッチ通知)**
- LINE通知を、テキストの羅列からカード型のリッチ表示(Flex Message)に変更
- 銘柄ごとに評価に応じた色付きヘッダーのカード、市場環境カード、スイング候補カードを表示
- 画像ホスティング不要でプライバシーも安全(保有データを公開URLに置かず、Broadcastで直接送信)

**フェーズ12: モメンタム指標の追加と生スコア順位付け**
- モメンタム(直近10日の株価変化率 = ROC)を指標に追加し、保有銘柄スコア・スイングスコアの両方に反映(短期の勢いを評価)
- 保有銘柄・スイング候補の並び順を、100点で頭打ちにしない内部の生スコアで決定(同点表示でも強い順に整列)

**フェーズ10: 要約通知(ルールベースの総合判断)**
- 出来高トレンド分析(直近の出来高が過去平均比で増加/減少しているか)と、株価の方向×出来高の方向を組み合わせた勢い判定を追加
- 全指標(テクニカル・ファンダメンタル・市場環境)を重み付けした総合スコアを算出し、評価記号(◎◎/◎/○/△/▲/×)に変換
- LINE通知は、詳細を全て内部で保持したうえで「総合評価・現在状況・判断理由(最重要3〜5個)・売買アクション・価格目安・重大リスク」のみに要約(30秒で判断できる視認性を優先)
- 詳細レポートは `python -m stock_analyzer.cli`(既定)、要約は `--summary` で確認可能。LINE通知は要約形式を使用

## セットアップ

pip install -r requirements.txt

## 使い方

```bash
python -m stock_analyzer.cli --portfolio portfolio.sample.csv
```

`portfolio.sample.csv` を参考に、自分の保有銘柄を記載したCSVを用意してください。

```csv
symbol,quantity,avg_cost,name,account
AAPL,10,150.00,アップル,特定
MSFT,5,300.00,マイクロソフト,NISA
```

`name` 列は任意です。記入するとその表記(日本語など好きな名前)で通知され、空欄の場合はyfinanceが返す企業名(多くは英語)が使われます。日本株の日本語名を表示したい場合はこの列に記入してください。

`account` 列も任意です。`NISA`(つみたて/成長投資枠を含む)と入力するとその銘柄を非課税口座として扱い、税効率の判断(NISAは温存、特定は損益通算・利益20万円以下は売却しやすい等)に反映します。空欄・未指定・`特定`・`一般` はすべて課税口座(特定)として扱います。

**保有していない銘柄も分析したい場合**は、`quantity`(数量)を空欄のまま `symbol` だけ入力してください。その行は「監視銘柄(未保有)」として、保有株と同じロジックで毎日分析・表示されます(🔍監視ラベル付き・保有損益や税は付きません)。数量を入力すればそのまま保有として扱われます。

## 検証と過学習対策(モデルの汎化)

目標は「バックテストで最も勝てる」ことではなく「**未知の将来相場でも期待値が高い**」こと。
そのため、指標・閾値・重みは**アウトオブサンプル(OOS)で有意なものだけ**を証拠付きで採用する。

**検証ハーネス(すべて価格/出来高/配当由来のみ。リーク無し)**
- `stock_analyzer/metrics.py` … 指標の単一の真実。EV/勝率/RR/PF/**Sharpe/Sortino/Calmar/MaxDD**/保有日数/分位点に加え、
  **ブートストラップ信頼区間(CI)**(同一エントリー日をブロック化)と**モンテカルロ**でドローダウン分布を出す。
- `stock_analyzer/validation.py` … **ローリング/拡張ウォークフォワード**、**purged/embargo 付き時系列CV**、
  **感度分析**、そして**採用ゲート**。重みは学習窓だけから導出し「学習日 < 検証日」を厳守する。
- `stock_analyzer/dynamic_thresholds.py` … RSI/BB/ATR等の固定 vs ボラ条件付き閾値(自由度は1〜2に制限)。
  動的化は検証で勝った場合のみ採用する。

**採用の流れ(証拠先行)**
1. `python -m stock_analyzer.research --period 10y --step 5` を実行(重い・要ネット)。
2. 各特徴量をローリングWFで検証し、`data/feature_analysis.json` に監査結果、
   `data/adopted_model.json` に**採用セット**(採用/棄却の根拠・OOS上乗せのCI・相場別の強弱・収縮した重み)を出力。
3. **採用ゲート**: OOS上乗せの**CI下限 > 0** かつ**複数相場で符号一致**かつ**感度が平坦**なものだけ `keep`。
   勝率ではなく**期待値(のCI下限)**で判断する。CIが出ない(標本不足)ものは採用しない。
4. `stock_analyzer/model_store.py` が重み/閾値の単一出所。**既定は現行値(フォールバック)で挙動不変**。
   採用セットを内容確認したうえで環境変数 `USE_ADOPTED_MODEL=1` を設定すると、棄却された特徴量がライブから外れる。
   ライブ(`summary`)とバックテスト(`backtest`)は同じ出所を参照する(パリティはテストで保証)。

**継続的な再評価**
- 定期的に `research` を再実行 → `adopted_model.json` を更新 → 内容を人が確認 → `USE_ADOPTED_MODEL` で有効化。
- `python -m stock_analyzer.verify_log` で、実運用の判断ログ(`data/judgment_log.csv`)と実現リターンを照合し、
  採用セットの有効性を継続チェックする。

**限界(誠実に明記)**
- 点在時点の**ファンダ/信用/ニュース/セクター資金流入**は無料データ源が無く、厳密検証の対象外。
  ライブの割安率・需給★等はヒューリスティックのまま(ラベル付き)。
- 標本は約10年・上昇相場に偏るため CI は広い。暴落/引締め相場は小標本＝低信頼。断定はしない。
- in-sample 成績は下がることがある。多くの候補や動的閾値がゲートで棄却されるのは、過学習防止が正しく働いている証。

## セキュリティ / 秘密情報の取り扱い

- **実際の保有銘柄データはコミットしないでください。** `portfolio.sample.csv` はサンプルのみです。自分の保有銘柄は `portfolio.csv` として作成してください(`.gitignore` で除外済み)。
- **APIキー・トークン(LINE Notifyのトークンなど)はコードに直接書かず、`.env` など `.gitignore` 対象のファイルや環境変数で管理してください。**
- このリポジトリはPublicのため、上記を守らないとコード・データが誰でも閲覧可能な状態で公開されます。

## Googleスプレッドシートの準備(保有銘柄の入力)

1. [Google Cloud Console](https://console.cloud.google.com/) で新規プロジェクトを作成
2. 「APIとサービス」→「ライブラリ」から **Google Sheets API** と **Google Drive API** を有効化
3. 「APIとサービス」→「認証情報」→「認証情報を作成」→「サービスアカウント」を作成(役割は付与不要)
4. 作成したサービスアカウント →「キー」タブ →「鍵を追加」→「新しい鍵を作成」で **JSON** 形式のキーをダウンロード
   - JSON内の `client_email` がサービスアカウントのメールアドレス
5. 保有銘柄を入力する。ゼロから作らず、同梱の整形済みテンプレート **[`portfolio_template.xlsx`](portfolio_template.xlsx)** を使うのが簡単です:
   - Googleスプレッドシートを開く → ファイル → インポート → アップロード → `portfolio_template.xlsx` を選択
   - 「保有銘柄」タブのサンプル行を、自分の銘柄で上書き(見出し行 `symbol,quantity,avg_cost,name` は変更しない)
   - 使い方の詳細は同ファイルの「使い方」タブに記載
   - ※自分で作る場合は、1行目に `symbol,quantity,avg_cost` の見出しを入れて入力(任意で `name` 列で日本語名、`account` 列で口座区分〈NISA/特定〉を指定できます)
   - 日本株は `symbol` に**証券コードの数字だけ**を入力すればOKです(例: `7203`)。末尾の `.T` は自動で補完されます。米国株はティッカー(例: `AAPL`)をそのまま入力してください
6. スプレッドシートの「共有」から、手順4のサービスアカウントのメールアドレスを**閲覧者**として追加(これ以外には非公開のまま)
7. スプレッドシートのURL `https://docs.google.com/spreadsheets/d/【ここがID】/edit` からIDを控える

## GitHub Actionsのセットアップ(定期実行 + Discord通知)

以下をリポジトリの Settings → Secrets and variables → Actions → New repository secret から登録してください(値はGitHubの画面上で直接入力し、他の場所に貼り付けないでください)。

| Secret名 | 内容 |
| --- | --- |
| `DISCORD_WEBHOOK_URL` | Discordチャンネルの Webhook URL(現在の通知先) |
| `GOOGLE_SERVICE_ACCOUNT_JSON` | ダウンロードしたサービスアカウントJSONキーの中身全体 |
| `GOOGLE_SHEET_ID` | 保有銘柄スプレッドシートのID |
| `LINE_CHANNEL_ACCESS_TOKEN` | (停止中)旧・LINE情報通知用。現在はワークフローで無効化 |

**現在の通知は Discord のみ**です。AIファンドマネージャーの結論(本日の結論・税・リバランス等)は Discord 通知に統合済みのため、旧・LINE情報通知は停止しています(`daily-report.yml` で `LINE_CHANNEL_ACCESS_TOKEN` の受け渡しをコメントアウト)。LINEを再開したい場合はその行のコメントを外し、Secretを登録すれば復活します(コードは非破壊で保持)。

**Discordを使う場合(通知数の上限なし・推奨)**
1. Discordでサーバーを用意(なければ「サーバーを追加」で作成)
2. 通知したいチャンネルの「⚙️ 編集」→「連携サービス」→「ウェブフック」→「新しいウェブフック」
3. 「ウェブフックURLをコピー」して、その値を GitHub Secrets の `DISCORD_WEBHOOK_URL` に登録

登録後は `.github/workflows/daily-report.yml` のcronで指定した時刻(**毎日 12:00・21:00 JST**)に自動実行されます。市場休場日(土日・祝日・年末年始)は自動でスキップされるため、実質的に取引日の1日2回の通知になります。手動で試したい場合はGitHubの Actions タブから `workflow_dispatch` で即時実行できます。以降、保有銘柄を変更したい場合はスプレッドシートを直接編集するだけで、次回の自動実行に反映されます。

> GitHub Actionsのスケジュールは負荷状況により数分〜十数分遅れて実行されることがあります(GitHubの仕様)。
